# 激光雷达探测器上位机系统需求文档

## 1. 文档信息

- 项目名称：32×32 像素激光雷达探测器上位机系统
- 文档版本：V0.1
- 编写日期：2026-06-13
- 运行形式：基于 HTML、CSS、JavaScript 的浏览器上位机
- 通信方式：FPGA 通过串口向电脑发送数据

## 2. 项目背景

当前激光雷达探测器由 FPGA 采集 32×32 像素芯片输出的数据，并通过串口发送至电脑。为完成芯片测试、TDC 性能验证、数据校准及后续光激励成像，需要开发一套浏览器上位机系统。

上位机第一阶段重点实现串口数据采集、电激励测试、TDC 直方图恢复和固定延迟偏差校准，并为后续光激励模式下的二维成像功能预留扩展能力。

## 3. 项目目标

1. 使用 HTML、CSS 和 JavaScript 实现可在电脑浏览器中运行的上位机。
2. 通过浏览器连接 FPGA 对应的串口，持续接收、解析并显示数据。
3. 支持 32×32 像素芯片的电激励和光激励两种工作模式。
4. 在电激励模式下采集 TDC 量化结果，并实时恢复和显示统计直方图。
5. 对 TDC 测量中存在的固定延迟、零点偏移或分段偏差进行校准。
6. 保存原始数据、校准参数和处理后的结果，便于后续分析与复现。
7. 为光激励模式下的飞行时间分析和二维成像预留数据接口与显示区域。

## 4. 系统范围

### 4.1 第一阶段

- 浏览器串口连接与参数配置。
- 串口数据接收、缓存、解析及异常检测。
- 电激励模式测试。
- TDC 数据实时统计和直方图显示。
- 固定延迟偏差标定与数据校准。
- 32×32 像素选择及分像素统计。
- 原始数据和结果导出。

### 4.2 后续阶段

- 光激励数据采集。
- 32×32 像素强度图、时间图或距离图显示。
- 飞行时间峰值提取和距离计算。
- 多帧累积、降噪和成像结果导出。

## 5. 系统总体架构

数据链路如下：

```text
STOP/END 或光信号
        ↓
32×32 像素芯片及 TDC
        ↓
       FPGA
        ↓ 串口
浏览器上位机（HTML/JavaScript）
        ↓
实时显示、直方图、校准、数据保存及后续成像
```

浏览器串口功能优先采用 Web Serial API 实现。系统主要面向支持该接口的桌面版 Chromium 浏览器，例如 Google Chrome 或 Microsoft Edge。页面需要在 `localhost` 或安全上下文中运行，串口授权必须由用户主动点击按钮触发。

## 6. 功能需求

### 6.1 串口连接

上位机应提供以下功能：

- 点击“选择串口”按钮，由用户选择 FPGA 对应的串口设备。
- 支持设置波特率、数据位、停止位和校验位。
- 支持连接、断开和重新连接。
- 显示当前连接状态及串口参数。
- 显示累计接收字节数、有效数据包数、错误包数和丢包数。
- 串口异常断开时给出明确提示，并允许重新连接。
- 连接前不应自动申请串口权限。

默认串口参数需在 FPGA 串口设计确定后填写，目前暂定为可配置项。

### 6.2 数据接收与解析

- 连续读取 FPGA 发送的字节流。
- 使用接收缓存处理半包、粘包和跨批次数据。
- 根据通信协议识别帧头、数据长度、数据内容和校验字段。
- 校验错误或数据长度异常时丢弃错误帧，并记录错误数量。
- 支持开始采集、暂停采集、停止采集和清空缓存。
- 支持显示最近接收到的原始数据，显示格式至少包括十六进制。
- 解析后的记录至少应包含：
  - 工作模式。
  - 帧序号或数据包序号。
  - 像素行号和列号，范围均为 0～31。
  - TDC 原始码或原始时间值。
  - 有效标志和异常标志。

### 6.3 工作模式

#### 6.3.1 电激励模式

电激励模式用于验证 TDC 测量是否准确。FPGA 向芯片提供 STOP 和 END 间隔信号，TDC 对两个信号之间的时间间隔进行量化，并将结果经 FPGA 和串口发送至上位机。

该模式应支持：

- 设置或记录 FPGA 产生的理论 STOP-END 时间间隔。
- 连续采集指定次数或持续指定时间。
- 查看单个像素、多个像素或全部像素的数据。
- 统计样本数、均值、标准差、最小值、最大值和峰值位置。
- 实时生成 TDC 原始结果直方图。
- 同时查看校准前和校准后的统计结果。

#### 6.3.2 光激励模式

光激励模式用于后续激光回波采集和成像。第一阶段可先完成模式入口和数据接口预留，后续至少应支持：

- 按 32×32 像素位置接收数据。
- 显示像素计数、峰值时间、飞行时间或距离的二维热力图。
- 点击某个像素查看其时间直方图。
- 支持多帧累积、单帧显示和连续刷新。
- 支持坏点屏蔽和无效像素标记。

### 6.4 直方图

- 横轴可选择 TDC 原始码、时间值或校准后的时间值。
- 纵轴表示对应区间内的累计次数。
- 支持设置直方图区间、分箱数量或分箱宽度。
- 支持实时更新和停止刷新。
- 支持清空当前统计结果。
- 支持选择单个像素、指定像素集合或全部像素。
- 支持显示均值、标准差和峰值位置。
- 支持缩放、悬停查看数值和导出图表。
- 大量数据到达时，图表刷新和串口接收应解耦，避免因频繁绘图造成串口丢数。

### 6.5 TDC 偏差校准

系统应支持对 TDC 的固定延迟和测量偏差进行标定。例如，当理论间隔达到或超过 100 ns 后，测量结果可能出现一个固定附加延迟。

#### 6.5.1 标定流程

1. 用户设置一组已知的 STOP-END 理论时间间隔。
2. 每个理论时间点采集指定数量的 TDC 样本。
3. 系统计算各时间点的均值、标准差及测量误差。
4. 根据数据确定校准模型和参数。
5. 保存校准参数，并在后续采集中自动应用。
6. 使用独立测试点验证校准后的误差。

#### 6.5.2 校准模型

基础校准模型应支持增益和零点偏移：

```text
t_corrected = a × t_raw + b
```

对于达到某个时间阈值后产生固定附加延迟的情况，应支持分段补偿：

```text
当 t_raw < T_threshold：
    t_corrected = a1 × t_raw + b1

当 t_raw ≥ T_threshold：
    t_corrected = a2 × t_raw + b2 - d_fixed
```

其中：

- `t_raw`：TDC 原始测量时间。
- `t_corrected`：校准后的时间。
- `a1`、`a2`：比例系数。
- `b1`、`b2`：零点偏移。
- `T_threshold`：出现附加延迟的阈值，例如待验证的 100 ns。
- `d_fixed`：阈值后需要扣除的固定延迟。

实际采用统一线性模型、固定偏移模型或分段模型，应由标定数据决定。

#### 6.5.3 校准参数管理

- 显示当前使用的校准模型和参数。
- 支持手动输入、自动计算、应用、停用和恢复默认参数。
- 支持全芯片统一校准。
- 为每个像素独立校准预留能力，共 1024 组像素参数。
- 校准参数至少包含版本、生成时间、适用模式、时间单位和采样条件。
- 支持将参数导出为 JSON，并从 JSON 文件重新导入。
- 原始数据必须保留，校准不得覆盖原始测量值。

### 6.6 32×32 像素管理

- 使用行列坐标标识像素，建议范围为 `row=0～31`、`col=0～31`。
- 可通过二维像素矩阵选择某个像素。
- 显示当前像素的编号、行列坐标、样本数和统计结果。
- 支持全选、取消全选和坏点屏蔽。
- 像素编号与行列坐标的映射规则必须与 FPGA 保持一致。

### 6.7 数据保存与导出

- 支持保存串口原始字节流，便于离线重新解析。
- 支持导出解析后的 CSV 数据。
- 支持导出直方图统计数据。
- 支持导出和导入校准参数 JSON 文件。
- 导出数据应包含采集时间、串口配置、工作模式、像素位置、理论时间、原始值和校准值。
- 文件名建议包含日期、模式和测试编号。

## 7. 页面设计建议

页面建议划分为以下区域：

1. **串口控制区**：串口选择、波特率、连接、断开和连接状态。
2. **采集控制区**：工作模式、开始、暂停、停止、清空及采集次数。
3. **像素选择区**：32×32 像素矩阵和当前像素信息。
4. **数据显示区**：原始数据、解析数据、收包统计和错误信息。
5. **直方图区域**：原始值与校准值直方图、分箱设置和统计指标。
6. **校准区域**：理论时间、校准点、模型、参数、应用及导入导出。
7. **成像区域**：为后续光激励二维热力图预留。

## 8. 串口通信协议建议

正式开发前，FPGA 与上位机必须确定统一的数据帧格式。建议使用带帧头、长度、序号和校验的二进制协议，避免仅靠换行符区分数据。

建议字段如下：

| 字段 | 建议长度 | 说明 |
| --- | ---: | --- |
| 帧头 | 2 字节 | 固定标识，例如 `0xAA 0x55` |
| 协议版本 | 1 字节 | 用于后续协议升级 |
| 数据类型/模式 | 1 字节 | 电激励、光激励、状态或命令响应 |
| 数据长度 | 2 字节 | 载荷长度 |
| 帧序号 | 2～4 字节 | 用于检测丢包 |
| 像素行号 | 1 字节 | 0～31 |
| 像素列号 | 1 字节 | 0～31 |
| TDC 数据 | 待定 | 原始码或时间值 |
| 状态标志 | 1 字节 | 有效、溢出、错误等状态 |
| 校验 | 2 字节 | 建议使用 CRC16 |

需要明确多字节整数采用大端还是小端格式。若一次数据包中包含多个像素或多个 TDC 结果，还需增加样本数量及重复数据区域定义。

## 9. 非功能需求

- 页面在持续采集过程中不应出现明显卡顿。
- 串口接收、协议解析和图表刷新应相互解耦。
- 长时间采集时应限制页面日志和内存缓存大小。
- 所有异常应给出可理解的提示，不应导致页面直接失效。
- 数据解析、单位换算和校准计算应使用统一的数据定义。
- 核心处理模块应便于使用离线模拟数据进行测试。
- 系统应避免因刷新页面或误操作而静默丢失尚未导出的重要数据，并在必要时给出提示。

## 10. 验收标准

### 10.1 串口功能

- 能够在支持 Web Serial API 的浏览器中选择并连接 FPGA 串口。
- 能够稳定接收连续数据，正确处理半包和粘包。
- 能够识别错误帧，并显示收包、丢包和校验错误统计。

### 10.2 电激励和直方图

- 能够根据串口数据恢复指定像素的 TDC 直方图。
- 能够正确显示样本数、均值、标准差、最小值、最大值和峰值。
- 能够清空数据并开始新一轮独立测试。
- 高速接收时，图表显示不得阻塞数据读取。

### 10.3 校准功能

- 能够使用已知理论间隔生成校准参数。
- 能够对新采集数据自动应用校准。
- 能够同时保留和导出原始值及校准值。
- 能够导出校准参数，并在重新加载页面后通过导入恢复。
- 校准后的允许误差范围由芯片指标和实际测试要求另行确定。

### 10.4 数据导出

- 能够导出原始数据、解析数据、直方图数据和校准参数。
- 导出的数据包含足够的测试条件，能够支持离线复现。

## 11. 待确认事项

以下信息需要在编码前或联调初期确定：

1. FPGA 串口的默认波特率、数据位、停止位和校验方式。
2. 串口数据采用二进制还是 ASCII 文本格式。
3. 完整的数据帧结构、帧头、长度、大小端和 CRC 算法。
4. TDC 原始数据位宽、编码方式、时间单位和最小时间分辨率。
5. FPGA 是逐样本发送、逐像素发送，还是整帧打包发送。
6. 32×32 像素的扫描顺序及像素编号映射关系。
7. STOP 与 END 的定义、先后关系及理论间隔的控制方法。
8. “100 ns 后固定延迟”对应的准确触发条件、延迟量和是否所有像素一致。
9. 校准是全芯片统一执行，还是需要每个像素独立执行。
10. 直方图默认范围、分箱宽度、单次采样数量和目标数据速率。
11. 光激励模式下 FPGA 输出的是事件时间、计数、峰值还是已处理距离。
12. TDC 溢出、无效测量、重复事件和丢包的状态编码方式。
13. 校准后的允许误差和最终验收指标。

## 12. 建议实施顺序

1. 确定串口协议，并制作 FPGA 模拟数据或串口回放数据。
2. 实现浏览器串口连接、字节流接收和协议解析。
3. 实现电激励模式、像素选择和实时直方图。
4. 实现原始数据与统计结果导出。
5. 采集多组已知间隔数据，确定固定延迟的实际规律。
6. 实现统一或分像素校准，并完成误差验证。
7. 扩展光激励模式和 32×32 二维成像。
