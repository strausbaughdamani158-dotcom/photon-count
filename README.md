# ArrayScope 电激励演示上位机

当前版本专用于电激励功能演示，暂不包含光激励。

项目现在包含三个独立页面：

- `index.html`：电激励计时测试。
- `photon-count.html`：32 × 32 光子计数测试与灰度成像。
- `single-range.html`：O12 前两个像素单点测距。

## 启动

```powershell
node server.mjs
```

使用桌面版 Chrome 或 Edge 打开：

```text
http://127.0.0.1:8000
```

光子计数页面：

```text
http://127.0.0.1:8000/photon-count.html
```

单点测距页面：

```text
http://127.0.0.1:8000/single-range.html
```

页面顶部可相互切换。

## 单点测距模式

单点测距页面只观察 `O12` 输出组的前两个 16-bit HEX 数据：

```text
O12: 0x0518 0x0519 0x0003 0x0003 ...
```

- 第 1 个字对应 `R0C0`，第 2 个字对应 `R0C1`。
- 每个字仍按两段式 TDC 规则解码为 TOF。
- `0x0003` 视为无效像素；两个像素都有效时才计入一条测距样本。
- 实际测试距离取两个像素 TOF 的平均值再做距离换算。
- 页面默认距离系数为 `149.896229 mm/ns`，即按光速往返 TOF 换算；如 FPGA 输出已经是距离等效值，可将系数改为 `1`。
- 支持固定延迟扣除，计算公式为：

```text
距离 = max(平均 TOF - 固定延迟, 0) × 距离系数
```

## 光子计数模式

光子计数页面沿用 16 路输出格式，每路包含相邻两行的 64 个 16-bit HEX 数据。16 路收齐后组成一帧 32 × 32 图像。

```text
O12: 0x001D 0x0025 ... 共 64 个
O34: 0x0105 0x011D ... 共 64 个
...
O3132: ... 共 64 个
```

- 每个字直接取芯片低 11-bit 数据作为光子计数。
- 低 3 bit 不再忽略。
- 光子计数范围为 `0~2047`。
- 数字模式在每个像素方块中显示光子数。
- 灰度模式将 11-bit 计数映射为灰度：`0` 为黑色，`2047` 为白色。
- 像素顺序沿用芯片蛇形输出：第一行 `C0~C31`，第二行 `C31~C0`。

计数演示采集采用与计时页面相同的串口门控：

- 未检测到串口输入时不生成计数帧。
- 检测到任意串口字节后，按页面设置的帧率生成完整 16 路数据。
- 默认帧率为 `100 Hz`，支持 `1~500 Hz` 调整。
- 串口输入停止约 `0.4 s` 后自动暂停，恢复输入后继续。
- 串口断开或数据流结束后立即停止。
- “芯片采集接收帧”显示最新完整帧，包含 `O12` 至 `O3132` 的全部数据。
- 每个模拟字直接编码完整的 11-bit 光子计数，不再拼接固定低位。

### START / END 计数窗口

计数窗口表示 START 与 END 信号上升沿之间的时间差，单位为 `μs`，默认值为 `10 μs`。

页面向 FPGA 发送 ASCII 命令：

```text
SET_COUNT_WINDOW 10.000us\r\n
```

FPGA 根据该时间产生 START/END 信号并送入芯片。只有命令发送成功后，新窗口才会成为当前生效值。模拟光子计数随窗口长度按比例变化。

### 计数图样与回卷

- 每次成功设置曝光窗口后都会重新建立多尺度光场，不会仅对上一幅图统一增减亮度。
- 曝光长度同时影响计数强度、局部响应和空间结构；不同曝光参数下不会保持完全相同的图样轮廓。
- 单帧图样由不规则多尺度结构、固定图样噪声、局部瞬态事件和光子散粒噪声组成。
- 相邻像素保留自然相关性，但相关尺度会随机变化，不采用固定周期或固定比例的全局平滑演进。
- 每帧还包含独立瞬态散斑、轻微行列读出扰动和非固定寿命的局部闪烁。
- 11-bit 计数器不进行饱和截断。
- 原始计数达到 `2048` 后按模 2048 回卷，例如 `2048 → 0`、`2348 → 300`。
- 较长的计数窗口会提高原始计数，并产生更多回卷区域。

### 坏点配置

- 光子计数页面每次加载都会以禁用缓存的方式读取根目录下的 `bad-pixels.json`。
- 配置采用零基坐标 `[row, col]`，当前固定包含 142 个坏点。
- 其中 57 个坏点位于靠近串口接口的最右 8 列 `C24~C31`，占全部坏点约 40%。
- 坏点计数始终强制为 `0`，对应串口字为 `0x0000`，灰度成像中始终显示为纯黑。

## 演示流程

1. 连接 FPGA 串口。
2. 设置理论 TOF 时间，并点击“设置 TOF 并发送”。
3. 上位机向 FPGA 发送 ASCII 命令：

```text
SET_TOF 100.00000ns\r\n
```

4. 上位机持续监听真实串口输入。
5. 未收到任何真实串口字节时，系统不会生成采集帧。
6. 收到真实串口字节后立即生成第一帧，默认按 `100 Hz` 连续采集；帧率可在页面修改。
7. 真实 RX 停止后约 `0.4 s` 自动暂停；恢复输入后继续。
8. 断开串口或串口数据流结束后，采集立即停止。

真实 TX/RX 调试卡片默认隐藏，可通过顶部“显示通信调试”开关显示。公开界面显示“芯片采集接收帧”。

## 芯片采集帧格式

电激励只使用 O1/2 输出组。一帧包含 64 个 16-bit HEX：

```text
O12: 0x0518 0x0519 0x0003 0x0003 ... 共 64 个
```

- 第 1 个字：R0C0 的有效 TDC 数据。
- 第 2 个字：R0C1 的有效 TDC 数据。
- 后续 62 个字：固定为无数据值 `0x0003`。
- 默认每秒生成 100 帧，可设置为 `1~500 Hz`。
- 芯片测量固定偏差为 `+15 ns`。
- 普通采集误差服从零均值高斯分布，并自动叠加少量长尾异常。
- 芯片性能等级是统计模型的唯一可调参数，范围为 `0%~100%`，默认 `60%`。
- 性能等级越低，主峰越宽、长尾概率越高、异常偏移越大，并出现更明显的帧间公共漂移。
- 主分布标准差、异常概率、异常分布宽度和帧间漂移由性能等级自动计算。
- 串口连接状态后的小数字表示当前性能值。
- 在页面任意位置长按鼠标左键一次增加 `10`，一次长按仅生效一次；双击鼠标右键一次减少 `10`。
- 性能值不在通信调试卡片中显示，仅在当前页面会话中生效；刷新后恢复为 `60`。
- 1 LSB 固定为 `0.15625 ns`。

## 两段式 TDC

11-bit 数据由 8-bit 粗计数和 3-bit 精计数组成：

```text
[高 8 bit 粗计数][低 3 bit 精计数]
```

粗计数低电平有效，因此：

```text
粗计数值 = NOT(高 8 bit)
粗计数时间 = 粗计数值 × 1.25 ns
精计数时间 = 低 3 bit × 0.15625 ns
总时间 = 粗计数时间 + 精计数时间
```

示例：

```text
0001_0011_010
NOT(0001_0011) = 1110_1100 = 236
粗计数时间 = 236 × 1.25 ns = 295 ns
精计数时间 = 2 × 0.15625 ns = 0.3125 ns
总时间 = 295.3125 ns
```

目标时间无法精确映射到 0.15625 ns 网格时，会量化到最近的可表示值。`0x0003` 被保留为空数据，不会用于有效像素。

## 校准与直方图

采集演示数据先按两段式 TDC 规则编码，再重新解码得到测量时间。

页面默认设置固定误差为 `15 ns`：

```text
直方图显示时间 = TDC 解码时间 - 固定误差
```

例如理论 TOF 为 `100 ns`：

- 未校准峰值约为 `115 ns`。
- 扣除 `15 ns` 后，峰值约为 `100 ns`。
- 性能为 `60%` 时，主分布标准差约为 `0.54 ns`，并叠加少量更远的采集异常值。

直方图分箱宽度固定为一个 LSB，即 `0.15625 ns`。
